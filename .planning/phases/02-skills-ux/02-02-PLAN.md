**YAML Frontmatter:**
```yaml
---
name: id3-clear
description: >
  Reset IDDD project. Removes all IDDD-generated files and folders,
  restoring the project to pre-IDDD state. Shows confirmation before deletion.
  Trigger: clear iddd, reset iddd, remove iddd, clean project
user-invocable: true
disable-model-invocation: true
allowed-tools: Read Glob Grep Bash Write Edit
---
```

Note: `disable-model-invocation: true` is set because this is a destructive action -- Claude should NOT auto-invoke this skill, only the user should trigger it explicitly via /id3-clear.

**Body structure (5 sections):**

**Section: "# IDDD Project Clear"**
Opening: "Remove all IDDD-generated files from this project, restoring it to pre-IDDD state. This is a destructive action that requires explicit confirmation."
Then: "Communicate with the user in their language."

**Section: "## Step 1: Verify IDDD Installation"**
Instructions:
1. Check if `specs/entity-catalog.md` exists OR `CLAUDE.md` exists in the project root (using Glob or Read tool)
2. If NEITHER exists: respond "No IDDD files found in this project. Nothing to clear." and stop
3. If at least one exists: continue to Step 2

**Section: "## Step 2: Scan Deletion Targets"**
Instructions:
1. Check existence of each known IDDD directory and file. Use Bash tool with `test -d` and `test -f` commands.
2. Build two lists: "directories that exist" and "files that exist"
3. Known IDDD directories (check each):
   - `specs/`
   - `docs/`
   - `steering/`
   - `hooks/`
   - `skills/`
   - `.claude/skills/`
   - `.claude/hooks/`
   - `.codex/skills/`
   - `.agents/skills/`
   - `.iddd/`
4. Known IDDD files (check each):
   - `CLAUDE.md`
   - `AGENTS.md`
5. Additionally, check for user-authored content in steering/:
   - Check if `steering/product.md` exists
   - Check if `steering/data-conventions.md` exists
6. Only include items that actually exist in the deletion list

**Section: "## Step 3: Show Warning and Confirm"**
Render the warning block following this exact format (from UI-SPEC.md Component 6):

```
(warning triangle symbol) WARNING: This action cannot be undone.

The following IDDD files and directories will be deleted:

Directories:
  - specs/
  - docs/
  - steering/                  (warning: contains user-authored content)
  - hooks/
  - skills/
  - .claude/skills/
  - .claude/hooks/
  - .codex/skills/
  - .agents/skills/
  - .iddd/

Files:
  - CLAUDE.md
  - AGENTS.md

(warning triangle symbol) Files with user-authored content:
  - steering/product.md        (your product vision)
  - steering/data-conventions.md (your data conventions)

Proceed? [y/N]
```

Rules for rendering:
- Only list directories and files that actually exist (from Step 2 scan)
- If `steering/` exists, add "(warning: contains user-authored content)" annotation
- The user-authored files section only appears if `steering/product.md` or `steering/data-conventions.md` actually exist
- Default is N (deny). Only proceed on explicit "y" or "yes" response.
- Wait for user response. Do NOT proceed without confirmation.

**Section: "## Step 4: Execute Deletion"**

On user confirmation ("y" or "yes"):
1. Delete all identified directories using Bash: `rm -rf specs/ docs/ steering/ hooks/ skills/ .claude/skills/ .claude/hooks/ .codex/skills/ .agents/skills/ .iddd/` (only the ones that exist)
2. Delete individual files using Bash: `rm -f CLAUDE.md AGENTS.md` (only the ones that exist)
3. Count how many directories and files were deleted
4. Show completion summary: "IDDD files removed. {N} directories and {M} files deleted. Project restored to pre-IDDD state."

On user denial ("n", "N", empty, or any other response):
- Show: "Clear cancelled. No files were removed."
- Stop.

**Important safety rules (include as a note at the end):**
- NEVER delete files outside the known IDDD file list
- NEVER use glob patterns like `rm -rf *` -- only delete specific known paths
- NEVER skip the confirmation step
- If the user asks to delete specific files only, inform them that /id3-clear removes all IDDD files. For selective deletion, they should use manual file operations.
